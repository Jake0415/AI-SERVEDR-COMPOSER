# 코드 리뷰 모드 컨텍스트

현재 **코드 리뷰 모드**입니다. 코드 변경사항을 체계적으로 검토합니다.

## 리뷰 체크리스트

### 정확성

- [ ] 로직 오류 없음
- [ ] 엣지 케이스 처리됨
- [ ] 에러 핸들링 적절함
- [ ] 올바른 타입 사용

### 보안

- [ ] 시크릿 하드코딩 없음
- [ ] 사용자 입력 검증됨
- [ ] XSS/SQL 인젝션 방지됨
- [ ] `dangerouslySetInnerHTML` 미사용 또는 검증됨

### 성능

- [ ] 불필요한 리렌더링 없음
- [ ] 적절한 메모이제이션 (필요 시)
- [ ] Server Component 적절히 활용
- [ ] 이미지 최적화 (`next/image`)

### 가독성

- [ ] 명확한 변수/함수명
- [ ] 적절한 코드 분할
- [ ] 불필요한 주석 없음
- [ ] 일관된 코딩 스타일

### 테스트

- [ ] 새 기능에 대한 테스트 포함
- [ ] 기존 테스트 통과
- [ ] 커버리지 기준 충족 (80%+)

### 한국어 로컬라이제이션

- [ ] 사용자 대면 텍스트 한국어 작성
- [ ] 날짜/시간 형식 적절
- [ ] 에러 메시지 한국어

## 심각도 수준

- **Critical**: 버그, 보안 취약점, 데이터 손실 위험 — 반드시 수정
- **Warning**: 성능 문제, 모범 사례 위반 — 수정 권장
- **Info**: 스타일 개선, 제안 사항 — 선택적 수정

## 출력 형식

```
## 코드 리뷰 결과

### Critical
- [파일:줄번호] [문제 설명]

### Warning
- [파일:줄번호] [문제 설명]

### Info
- [파일:줄번호] [개선 제안]

### 요약
변경 파일: N개 | Critical: N건 | Warning: N건 | Info: N건
```
