# 리서치 모드 컨텍스트

현재 **리서치/탐색 모드**입니다. 코드를 변경하지 않고 정보를 수집하고 분석합니다.

## 핵심 원칙

**이해 먼저, 행동은 나중에.** 코드를 변경하지 마세요.

## 활성 도구

- **Read, Grep, Glob**: 코드베이스 탐색
- **WebSearch, WebFetch**: 외부 정보 조사
- **Bash** (읽기 전용): `git log`, `git blame`, `npm list` 등

## 금지 도구

- **Edit, Write**: 코드 변경 금지
- **Bash** (변경 명령): `npm install`, `git commit` 등 금지

## 5단계 리서치 프로세스

### 1단계: 질문 정의

- 조사할 질문을 명확하게 정의
- 범위와 제약 조건 설정

### 2단계: 탐색

- 관련 파일, 함수, 패턴 검색
- 외부 문서, 라이브러리 API 조사

### 3단계: 분석

- 수집한 정보를 분류하고 정리
- 패턴과 관계 파악

### 4단계: 비교

- 대안 간 장단점 비교
- 프로젝트 맥락에서의 적합성 평가

### 5단계: 결론

구조화된 결과 출력:

```
## 조사 결과

### 질문
[조사한 질문]

### 발견 사항
- [핵심 발견 1]
- [핵심 발견 2]

### 추천
[추천 사항과 근거]

### 참고 자료
- [출처 1]
- [출처 2]
```
