# /plan - 기능 기획

기능 구현 전 구조화된 계획을 수립합니다. 이 커맨드는 `planner` 에이전트의 워크플로우를 실행합니다.

> **참고**: PRD 작성은 `/prd-generate`, 로드맵 생성은 `development-planner` 에이전트를 사용하세요.

## 입력

기획할 기능: $ARGUMENTS

## 실행 절차

1. **프로젝트 구조 파악**: 현재 파일 구조와 기존 코드를 분석합니다.

2. **요구사항 분석**: 입력된 기능 요구사항을 명확하게 재정의합니다.

3. **영향 파일 식별**: 변경이 필요한 파일과 새로 생성할 파일을 정확한 경로로 나열합니다.

4. **단계별 분해**: 구현을 독립적인 단계로 나누고, 각 단계에 다음을 포함합니다:
   - 구체적 작업 설명
   - 대상 파일 경로
   - 복잡도 (높음/중간/낮음)
   - 선행 의존성

5. **리스크 식별**: 잠재적 문제와 완화 전략을 제시합니다.

6. **구조화된 계획 출력**: 위 내용을 정리된 형식으로 출력합니다.

## 중요

- 코드 생성 전 **반드시 사용자 승인**을 받으세요
- 사용자가 "진행", "승인", "시작"이라고 말하기 전까지 코드를 작성하지 마세요
- CLAUDE.md의 코딩 규칙, 보안 규칙, 프로젝트 구조 규칙을 반드시 준수하세요
- 한국어 사용자 대면 텍스트를 고려하세요
