# /checkpoint - 개발 체크포인트

현재 작업 상태를 체크포인트로 저장합니다.

## 입력

체크포인트 설명: $ARGUMENTS

## 실행 절차

### 1. 현재 상태 확인

```bash
git status
git diff --stat
```

변경된 파일 목록과 변경량을 확인합니다.

### 2. 품질 검사 (빠른 검증)

```bash
npx tsc --noEmit
npm run lint
```

타입 오류나 린트 오류가 있으면 알려주고 계속 진행할지 확인합니다.

### 3. 체크포인트 생성

변경사항을 스테이징하고 체크포인트 커밋을 생성합니다:

```bash
git add -A
git commit -m "🔖 체크포인트: [설명] (YYYY-MM-DD HH:mm)"
```

### 4. 요약 출력

```
=== 체크포인트 저장 완료 ===
설명: [체크포인트 설명]
커밋: [커밋 해시]
변경 파일: N개
시간: YYYY-MM-DD HH:mm
==========================
```

## 주의사항

- 체크포인트는 로컬 커밋이며 자동으로 push하지 않습니다
- 사용자에게 커밋 메시지를 확인받은 후 진행하세요
- `.env.local` 등 시크릿 파일이 포함되지 않았는지 확인하세요
