# /analyze-service - 서비스 분석

서비스/제품을 체계적으로 분석하여 전략적 인사이트를 도출합니다. 이 커맨드는 `service-analyst` 에이전트의 분석 프레임워크를 실행합니다.

> **다른 도구와의 차이**: `/research-market`은 시장 전체의 규모/성장률/트렌드를 조사합니다. `/analyze-service`는 특정 서비스의 경쟁/사용자/SWOT를 분석합니다.

## 입력

분석 대상: $ARGUMENTS

## 분석 절차

### 1단계: 서비스 개요 파악

분석 대상 서비스/제품의 기본 정보를 수집합니다:

- 핵심 가치 제안 (Value Proposition)
- 타겟 시장 및 사용자
- 비즈니스 모델 (수익 구조)
- 핵심 기능과 차별화 포인트

WebSearch와 WebFetch를 활용하여 공개 정보를 수집하세요.

### 2단계: 경쟁 환경 분석

직접/간접 경쟁사를 식별하고 비교합니다:

| 항목 | 분석 대상 | 경쟁사 1 | 경쟁사 2 | 경쟁사 3 |
|------|----------|---------|---------|---------|
| 핵심 기능 | | | | |
| 타겟 사용자 | | | | |
| 가격 정책 | | | | |
| 강점 | | | | |
| 약점 | | | | |

포지셔닝 맵을 작성하여 시장 내 위치와 기회 영역을 시각화합니다.

### 3단계: 사용자 분석

타겟 사용자를 세그먼트별로 분석합니다:

- **페르소나 정의**: 역할, 목표, 페인포인트, 현재 대안
- **적합성 스코어**: 각 세그먼트의 적합도 (1-10)
- **사용자 여정**: 인지 → 고려 → 가입 → 온보딩 → 활성 사용 → 충성/이탈
- 각 단계의 터치포인트, 감정, 기회 영역 식별

### 4단계: SWOT 분석

| | 긍정적 | 부정적 |
|---|--------|--------|
| **내부** | 강점 (Strengths) | 약점 (Weaknesses) |
| **외부** | 기회 (Opportunities) | 위협 (Threats) |

전략 매트릭스 (SO/WO/ST/WT 전략) 도출

### 5단계: 기회 영역 도출

식별된 기회를 Impact-Effort 매트릭스로 평가하고 우선순위를 매깁니다:

- **Quick Wins**: 높은 임팩트 + 낮은 노력
- **전략적 투자**: 높은 임팩트 + 높은 노력
- **후순위**: 낮은 임팩트

## 출력 형식

```
═══════════════════════════════
  서비스 분석 리포트: [서비스명]
═══════════════════════════════

## Executive Summary
[3-5줄 핵심 인사이트]

## 1. 서비스 개요
[가치 제안, 비즈니스 모델]

## 2. 경쟁 분석
[비교표, 포지셔닝]

## 3. 사용자 분석
[페르소나, 사용자 여정]

## 4. SWOT 분석
[4분면 + 전략 매트릭스]

## 5. 기회 영역
[우선순위 기반 추천]

## 6. 다음 단계
[구체적 액션 아이템]
```

## 주의사항

- 코드 변경 없이 분석에만 집중합니다
- 데이터에 기반한 객관적 분석을 제공합니다
- 추측이 필요한 부분은 명시적으로 가정(Assumption)임을 표기합니다
