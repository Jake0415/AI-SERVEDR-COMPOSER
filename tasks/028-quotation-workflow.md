# Task 028: 견적서 발행 워크플로우 구현

## 상태
- [x] ✅ 완료

## 개요
견적서의 전체 발행 프로세스(작성→검토→승인→발행→전달)를 구현한다. 견적번호 자동 채번, 유효기간, 납품/결제 조건, 개정(버전) 관리, 발행사+거래처 정보 자동 삽입, 직인 날인을 포함한다. F007 확장.

## 관련 PRD 기능
- [F007] 견적서 발행 및 출력 (확장)
- [F008] 낙찰 결과 기록 (상태 확장)
- [F015] 견적 공유 및 협업 (승인 워크플로우)
- [F017] 시스템 설정 (채번 규칙, 기본 조건)

## 의존성
- Task 027 (거래처 관리) 완료 필요
- Task 019 (견적서 출력) 완료 필요

## 관련 파일
- `app/api/quotations/[id]/publish/route.ts` — 발행 API
- `app/api/quotations/[id]/revise/route.ts` — 개정 API
- `app/api/quotations/[id]/approve/route.ts` — 승인 API
- `lib/services/quotation-number.ts` — 견적번호 채번 서비스
- `lib/services/quotation-pdf.ts` — 견적서 PDF 생성 (확장)
- `components/quotation/quotation-form.tsx` — 견적서 작성 폼

## 수락 기준
- [ ] 견적번호가 설정된 규칙에 따라 자동 채번된다
- [ ] 유효기간이 기본 30일로 자동 설정되고 수정 가능하다
- [ ] 납품조건/결제조건을 입력할 수 있다
- [ ] 견적서 상태가 draft→review→approved→published 순서로 전환된다
- [ ] 발행 시 발행사 정보+거래처 정보+직인이 자동 삽입된다
- [ ] 견적서 개정(Rev.1, Rev.2) 시 원본과 연결되어 이력이 유지된다
- [ ] 유효기간 만료 시 상태가 자동으로 expired로 변경된다
- [ ] 품목에 H/W 외 S/W, 서비스, 유지보수를 추가할 수 있다

## 구현 단계

### 견적번호 채번
- [ ] 견적번호 자동 채번 서비스 구현
  - 테넌트의 quotation_prefix + 날짜 + 일련번호 조합
  - 예: Q-20260318-001
  - 개정 시: Q-20260318-001-R1
- [ ] 시스템 설정에서 채번 규칙 변경 가능 (접두사, 구분자 등)

### 견적서 작성 폼 확장
- [ ] 거래처 선택 필드 (검색+드롭다운, 신규 등록 버튼)
- [ ] 유효기간 입력 (기본값: 테넌트 설정의 default_validity_days)
- [ ] 납품조건 입력 (현장설치도/납품장소도/상차도 선택 또는 직접 입력)
- [ ] 납기일 입력
- [ ] 결제조건 입력 (거래처 기본값 자동 삽입, 수정 가능)
- [ ] 비고/특기사항 입력

### 품목 구분 확장
- [ ] quotation_items에 item_type 필드 활용 (hardware/software/service/maintenance)
- [ ] H/W: 기존 부품 DB에서 선택
- [ ] S/W: 품명/규격/단가 직접 입력 (OS 라이선스, DB 라이선스 등)
- [ ] 서비스: 설치/구축/마이그레이션 등 직접 입력
- [ ] 유지보수: 연간 유지보수 비율(H/W의 8~15%, S/W의 15~22%) 자동 계산 옵션

### 승인 워크플로우
- [ ] 상태 전환 API: draft → review → approved → published
- [ ] review 요청 시 검토자에게 알림 (F016 연동)
- [ ] approved 후 발행 시 직인 이미지 자동 삽입
- [ ] 권한 체크: 작성(전원), 승인(관리자 이상), 발행(관리자 이상)

### 견적서 개정
- [ ] 발행 후 수정 필요 시 "개정" 버튼 → 새 버전 생성 (revision +1)
- [ ] parent_quotation_id로 원본과 연결
- [ ] 개정 이력 목록 표시 (원본 → Rev.1 → Rev.2)
- [ ] 이전 버전은 읽기 전용

### 견적서 PDF 생성 확장
- [ ] 발행사 정보 자동 삽입 (테넌트 데이터에서)
- [ ] 거래처 정보 자동 삽입 (선택된 거래처에서)
- [ ] 직인/도장 이미지 삽입
- [ ] 견적번호, 유효기간, 납품/결제 조건 표시
- [ ] 품목을 H/W, S/W, 서비스, 유지보수 구분하여 소계 표시
- [ ] 공급가 + VAT(10%) + 총액 표시

### 유효기간 관리
- [ ] Vercel Cron Job으로 만료 견적 자동 상태 변경 (daily)
- [ ] 만료 3일 전 알림 (F016 연동)

## 테스트 체크리스트
- [ ] 견적번호 채번 → 중복 없이 순차 생성 확인
- [ ] draft → review → approved → published 전체 플로우 확인
- [ ] 개정 생성 → 원본 연결 + 버전 번호 증가 확인
- [ ] PDF 출력 → 발행사/거래처/직인/품목/금액 정확성 확인
- [ ] 유효기간 만료 → 상태 자동 변경 확인
- [ ] 멤버 → 승인/발행 불가 확인
