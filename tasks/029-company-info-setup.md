# Task 029: 발행사(자사) 정보 관리 UI 구현

## 상태
- [ ] 미완료

## 개요
테넌트의 확장된 회사 정보(대표자, 주소, 업태/종목, 로고, 직인, 계좌 등)를 입력/수정하는 UI를 구현한다. 초기 설정 페이지와 시스템 설정 페이지에서 관리. F011/F017 확장.

## 관련 PRD 기능
- [F011] 테넌트 생성 (초기 설정 시 회사 정보 입력)
- [F017] 시스템 설정 (회사 정보 수정)

## 관련 파일
- `app/setup/page.tsx` — 초기 설정 페이지 (회사 정보 입력 확장)
- `app/(dashboard)/settings/page.tsx` — 시스템 설정 페이지 (회사 정보 탭)
- `app/api/tenant/route.ts` — 테넌트 정보 수정 API
- `app/api/tenant/logo/route.ts` — 로고 업로드 API
- `app/api/tenant/seal/route.ts` — 직인 업로드 API

## 수락 기준
- [ ] 초기 설정 시 필수 회사 정보(회사명, 사업자번호, 대표자, 주소, 업태, 종목)를 입력할 수 있다
- [ ] 시스템 설정에서 회사 정보를 수정할 수 있다
- [ ] 로고 이미지를 업로드/변경/삭제할 수 있다
- [ ] 직인/도장 이미지를 업로드/변경/삭제할 수 있다
- [ ] 견적 기본 설정(유효기간, 결제조건, 채번 접두사)을 변경할 수 있다
- [ ] 입금 계좌 정보를 등록/수정할 수 있다

## 구현 단계
- [ ] 테넌트 정보 수정 API 구현 (PUT /api/tenant)
- [ ] 로고 업로드 API (Supabase Storage, 이미지 리사이징)
- [ ] 직인 업로드 API (투명 배경 PNG 권장 안내)
- [ ] 초기 설정 페이지 확장: Step 2에 업태/종목/주소/대표자 추가
- [ ] 시스템 설정 페이지 > 회사 정보 탭 UI
  - 기본 정보 폼 (회사명, 사업자번호, 대표자, 주소, 업태, 종목)
  - 연락처 폼 (전화, 팩스, 이메일)
  - 로고/직인 업로드 영역 (미리보기, 삭제)
  - 계좌 정보 폼 (은행, 계좌번호, 예금주)
  - 견적 설정 폼 (기본 유효기간, 결제조건, 채번 접두사)
- [ ] 견적서 PDF에 로고/직인 삽입 테스트

## 테스트 체크리스트
- [ ] 초기 설정 → 회사 정보 입력 → 테넌트 저장 확인
- [ ] 시스템 설정 → 정보 수정 → 저장 확인
- [ ] 로고 업로드 → 미리보기 → PDF 반영 확인
- [ ] 직인 업로드 → 투명 배경 → PDF 반영 확인
- [ ] 슈퍼어드민만 접근 가능 확인
