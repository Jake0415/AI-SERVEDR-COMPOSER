# Task 045: 수동 조립 - 스토리지 베이 시각화

## 개요

수동 조립 Step 4에서 스토리지 베이를 행렬 레이아웃으로 시각화합니다. 2.5인치 베이와 3.5인치 베이를 분리 표시하고, 베이 클릭 시 호환 드라이브(NVMe/SATA/SAS) 목록을 표시하며, 총 스토리지 용량을 실시간으로 계산합니다.

## 관련 기능

- **F024**: 수동 서버 조립 — 스토리지 베이 시각화 및 드라이브 선택
- **F025**: 호환성 매트릭스 확장 — C015(베이 초과), C016(폼팩터 불일치) 실시간 검증

## 현재 상태

- Task 043에서 베이스 서버 선택 완료 → 섀시의 drive_bays_25, drive_bays_35 정보 가용
- Task 040에서 메인보드의 supported_storage_interfaces 정보 가용
- 스토리지 베이 시각화 UI 없음
- Task 041의 호환성 규칙 C015, C016 활용 가능

## 수락 기준

- [ ] 2.5인치 베이와 3.5인치 베이가 분리된 행렬 레이아웃으로 표시됨
- [ ] 빈 베이(회색), 채워진 베이(녹색) 시각 구분
- [ ] 베이 클릭 시 호환 드라이브 목록(NVMe/SATA/SAS 필터링) 표시
- [ ] 총 스토리지 용량 실시간 표시
- [ ] C015, C016 규칙 위반 시 실시간 경고
- [ ] `tsc --noEmit` + `npm run build` 통과

## 구현 단계

### Step 1: 스토리지 베이 다이어그램 컴포넌트

- [ ] `components/assembly/storage-bay-diagram.tsx` (신규)
- [ ] 행렬 레이아웃:
  - 2.5인치 베이 섹션: 그리드 형태 (예: 4열 × n행)
  - 3.5인치 베이 섹션: 그리드 형태 (예: 4열 × n행)
  - 각 섹션에 라벨: "2.5\" 베이 ({used}/{total})" / "3.5\" 베이 ({used}/{total})"
- [ ] 베이 상태별 시각 구분:
  - 빈 베이: 회색 배경 + "비어 있음" 텍스트
  - 채워진 베이: 녹색 배경 + 드라이브 모델명/용량/인터페이스 라벨
- [ ] 베이 hover 시 툴팁 (장착된 드라이브 상세 또는 "비어 있음")

**관련 파일:**
- `components/assembly/storage-bay-diagram.tsx` (신규)
- `components/assembly/storage-bay.tsx` (신규 — 개별 베이 컴포넌트)

### Step 2: 드라이브 선택 패널

- [ ] `components/assembly/drive-selector-panel.tsx` (신규)
- [ ] 베이 클릭 시 사이드 패널(Sheet) 오픈:
  - 선택한 베이 위치 표시 (예: "2.5\" 베이 #5")
  - 호환 드라이브 목록 표시
- [ ] 드라이브 필터링:
  - 인터페이스 필터: NVMe / SATA / SAS (메인보드 supported_storage_interfaces 기반)
  - 폼팩터 필터: 해당 베이 크기에 맞는 드라이브만 표시 (2.5" 베이 → 2.5" 드라이브)
  - 용량 필터: 범위 슬라이더 (예: 480GB ~ 15.36TB)
  - 카테고리 필터: SSD / HDD
- [ ] 드라이브 카드: 모델명, 제조사, 인터페이스, 용량, 읽기/쓰기 속도, 가격
- [ ] 드라이브 선택 시 해당 베이에 즉시 반영

**관련 파일:**
- `components/assembly/drive-selector-panel.tsx` (신규)

### Step 3: 일괄 채우기 기능

- [ ] "일괄 채우기" 버튼:
  - 동일 드라이브로 여러 베이를 한번에 채우기
  - 채울 베이 수량 입력 (예: "NVMe SSD 8개를 2.5\" 베이에 장착")
  - 빈 베이에 순차적으로 자동 배치
- [ ] "모두 비우기" 버튼: 모든 베이를 비움 (확인 다이얼로그 포함)

**관련 파일:**
- `components/assembly/storage-bulk-fill.tsx` (신규)

### Step 4: 호환성 검증 및 요약

- [ ] 드라이브 배치 변경 시 실시간 호환성 검증:
  - C015: 베이 초과 시 빨간 배지 + 메시지
  - C016: 폼팩터 불일치 시 빨간 배지 + 메시지
- [ ] 스토리지 요약 정보 표시:
  - 총 스토리지 용량: {current}TB
  - 사용 베이: 2.5" {used}/{total}, 3.5" {used}/{total}
  - 인터페이스별 분포: NVMe {n}개, SATA {n}개, SAS {n}개
  - 총 스토리지 비용: ₩{total}

**관련 파일:**
- `components/assembly/storage-summary.tsx` (신규)
